# 체인지로그 (CHANGELOG)

온그린(On Green) 앱의 모든 주요 변경 사항을 이 파일에 기록합니다.

규칙은 [Keep a Changelog](https://keepachangelog.com/ko/1.1.0/) 형식을 따르며,
**기능이 추가될 때마다 버전을 올리고(`app.js`의 `APP_VERSION`) 여기에 항목을 남깁니다.**

> **버전 두 종류 주의**
> - `APP_VERSION` (`app.js`): 프런트엔드 앱 버전. 이 체인지로그가 추적하는 값이며 기능 추가마다 올립니다.
> - `API.VERSION` (`api.js`): 서버(Apps Script)와의 통신 동기화용 버전. 백엔드를 함께 고칠 때만 백엔드 `VERSION`과 맞춰서 바꿉니다.

버전 표기: `vMAJOR.MINOR.PATCH`
- **MAJOR** — 큰 구조 변경 / 호환이 깨지는 변경
- **MINOR** — 새 기능 추가
- **PATCH** — 버그 수정 · 소소한 개선

## [v12.2.1] - 2026-06-22

### 수정됨
- **서비스 워커가 배포 중 GitHub 404 페이지를 그대로 노출하던 문제 수정.**
  여러 변경을 연달아 배포하면 GitHub Pages가 재배포되는 짧은 순간 "There
  isn't a GitHub Pages site here" 404(HTTP 404)를 돌려주는데, 기존 서비스
  워커는 `fetch`가 *성공*했지만 응답이 404/5xx인 경우 캐시로 폴백하지 않고
  그 에러 페이지를 사용자에게 그대로 보여줬습니다(`try/catch`는 네트워크가
  끊겨 throw될 때만 폴백). 이제 네트워크 응답이 정상(`ok`)이 아니면 캐시에
  저장된 정상 화면으로 폴백하므로, 배포 중에도 홈스크린 앱이 깨지지 않습니다.

## [v12.2.0] - 2026-06-22

### 추가됨 (Added)
- 서비스 워커(`sw.js`) 도입 — 사파리 홈스크린(standalone) 앱에서 캐시 때문에 새 버전이 안 보이던 문제 해결.
  - network-first 전략: 온라인이면 항상 서버에서 최신 자산을 받아오고, 오프라인일 때만 캐시로 폴백.
  - 배포(버전 변경)할 때마다 옛 캐시를 자동으로 비우고 새 워커가 즉시 교체됩니다.
- API 호출(외부 출처)은 서비스 워커가 건드리지 않아 통신 동작에는 영향 없음.

### 변경됨 (Changed)
- `index.html` 의 자산 참조(`api.js`/`app.js`/`style.css`)에 버전 쿼리스트링(`?v=`)을 붙여 HTTP 캐시도 우회.
