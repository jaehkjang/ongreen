# 체인지로그 (CHANGELOG)

온그린(On Green) 앱의 모든 주요 변경 사항을 이 파일에 기록합니다.

규칙은 [Keep a Changelog](https://keepachangelog.com/ko/1.1.0/) 형식을 따르며,
**기능이 추가될 때마다 버전을 올리고(`app.js`의 `APP_VERSION`) 여기에 항목을 남깁니다.**

> **버전 두 종류 주의**
> - `APP_VERSION` (`app.js`): 프런트엔드 앱 버전. 이 체인지로그가 추적하는 값이며 기능 추가마다 올립니다.
> - `API.VERSION` (`api.js`): 서버(Apps Script)와의 통신 동기화용 버전. 백엔드를 함께 고칠 때만 백엔드 `VERSION`과 맞춰서 바꿉니다.

버전 표기: `vMAJOR.MINOR.PATCH`
- **MAJOR** — 큰 구조 변경 / 호환이 깨지는 변경
- **MINOR** — 새 기능 추가
- **PATCH** — 버그 수정 · 소소한 개선

### 추가됨
- **업데이트 안내 팝업.** 앱이 새 버전으로 업데이트되면, 다음 접속 때 **바뀐 내용이 팝업으로 한 번** 뜹니다(관리자도 동일). 닫으면 같은 버전에선 다시 뜨지 않습니다. 팝업 위·아래에 **현재 버전 번호를 작게** 함께 표시합니다.

  - **알림 배지 연동** — 업데이트가 있으면 홈 📢 아이콘에 빨간 알림이 뜨고(`updatePending()`이 `APP_VERSION` 기준으로 판정), **공지 게시판을 한 번 열면 사라집니다**(`markNoticesSeen`이 `og_update_seen`을 현재 버전으로 기록).
  - 상태 키 분리: 첫 설명서 팝업은 종전대로 `og_guide_seen`, 업데이트 팝업 1회 표시는 `og_update_popped`, 배지(읽음 처리)는 `og_update_seen`. 첫 로그인 사용 설명서 팝업과 충돌하지 않습니다(첫 설치 땐 설명서만 뜨고 업데이트 팝업은 생략).

> 프런트엔드 전용 변경입니다. `Apps_Script.gs`를 건드리지 않았으므로 `api.js`의 `API.VERSION`은 그대로 둡니다.

- **🍩 퍼팅 등급 판정을 "라운드 총 퍼팅"에서 "GIR홀 퍼팅(순수 퍼팅력)"으로 교체.** 총 퍼팅은 GIR이 낮으면 칩샷 후 원퍼팅으로 오히려 줄고, GIR이 높으면 먼 첫 퍼트가 많아 늘어나는 교란이 있었습니다. 정규로 올린 홀의 홀당 퍼팅(기준값 🟢 `gpGood` 1.9 / 🟡 `gpBad` 2.05개/홀)으로 판정해 그린 적중률과 분리했습니다. GIR홀이 하나도 없는 경우에는 기존 총 퍼팅 기준(`puttGood`/`puttBad`)으로 자동 폴백합니다. 카드에는 GIR홀 퍼팅·총 퍼팅·3퍼팅을 함께 표시합니다.
- 설정 → 📊 분석 기준에 숏게임·GIR홀 퍼팅 기준값 입력 항목 추가, 설명문·통계 가이드도 새 지표에 맞춰 갱신. 기준값 입력칸은 소수 입력(GIR홀 퍼팅)을 위해 `inputmode="decimal"`로 변경.

> 프런트엔드 전용 변경입니다. `Apps_Script.gs`를 건드리지 않았으므로 `api.js`의 `API.VERSION`은 그대로 둡니다. (새 BENCH 키 `gpGood`/`gpBad`/`scrGood`/`scrBad`는 기존 BENCH 저장/병합 경로로 그대로 직렬화되며, 서버에 값이 없으면 위 기본값이 유지됩니다.)

- 서비스 워커(`sw.js`) 도입 — 사파리 홈스크린(standalone) 앱에서 캐시 때문에 새 버전이 안 보이던 문제 해결.
  - network-first 전략: 온라인이면 항상 서버에서 최신 자산을 받아오고, 오프라인일 때만 캐시로 폴백.
  - 배포(버전 변경)할 때마다 옛 캐시를 자동으로 비우고 새 워커가 즉시 교체됩니다.
- API 호출(외부 출처)은 서비스 워커가 건드리지 않아 통신 동작에는 영향 없음.

### 변경됨 (Changed)
- `index.html` 의 자산 참조(`api.js`/`app.js`/`style.css`)에 버전 쿼리스트링(`?v=`)을 붙여 HTTP 캐시도 우회.
